60W佳缘数据抓取-存储-展示

2018-01-24

文章目录

Introduction

  我们抓取了世纪佳缘近60W数据(仅个人学习使用)，一开始使用mongodb存储，由于机器配置问题，数据迁移到了mysql；然后使用nginx+flask搭建服务器提供数据接口；前端使用小程序进行展示。

  下面逐一介绍下各环节实现。

python数据抓取

  分析佳缘网站，可以通过一个接口+各种筛选参数获取用户数据，但是翻页到几十页后会很大概率出现重复数据，也就是同一筛选条件不会抓全，所以使用各种筛选参数的组合，抓取尽可能多的数据。另外需要登录下，才能获取接口数据，使用python session模拟登陆。

  前期使用mongodb直接存储接口json数据，无需格式化处理。

  通过一段时间的抓取，累计了近60W用户数据。

  部分代码如下：

def search_con():
    #con函数返回各种筛选参数组合数组
    arr_con = con()
    for each in arr_con:
        for page in xrange(1, 100):
            data = {
                'p': page,
                'sex': 'f',
                'stc': each,
            }
            print 'current_page:{}'.format(page)
            response = session.post('接口api',
                                    headers=SEARCH_HEADERS, data=urlencode(data))
            if not response.ok:
                print 'page:{} failed'.format(page)
                continue

            users = response.json()['userInfo']
            if not users:
                print 'get emtpy data'
                break;
            for user in users:
                dt = mongo_collection.find_one({'uid': user['uid']})
                if not dt:
                    mongo_collection.insert(user)
                else:
                    print '{} had existed.'.format(user['uid'])

mongodb数据迁移至mysql

  1. 将MongoDB里的数据导出到csv文件，mongo自带工具mongoexport
/your_mongodb_path/bin/mongoexport -h ip(192.168.0.102) -u 登录帐号 -p 库登录密码 -d 数据库名称 -c 数据库集合名 -f _id,字段1,字段2 --type=csv -o 保存路径(/tmp/records.csv)
  1. 将csv文件导入到MySQL数据库中，可以用MySQL的load命令。
load data local infile '/tmp/records.csv'  into table `records` character set utf8 fields terminated by ',' optionally enclosed by '"' lines terminated by '\n'  ignore 1 lines;

注意：mysql导入文件时，遇权限问题，登陆使用 mysql --local-infile -u user -ppasswd

  通过这种方式导入，可能会遇到了字段乱序或者乱码的问题，最后通过程序逐行导入。
